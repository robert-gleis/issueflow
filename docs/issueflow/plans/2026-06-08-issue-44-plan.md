# Merge Readiness Check Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a merge readiness gate that blocks `issueflow merge` until verification, review, PR, and candidate-branch checks pass, with PR-visible checklist comments and re-evaluation after fixes.

**Architecture:** Merge dependency branches (#29 verification gate, #43 PR creation) into this worktree first. Add pure `evaluateMergeReadiness` in `src/integration/`, Zod-validated `merge-readiness.json` + `merge:*` GitHub labels, PR comment sync, and `issueflow merge evaluate|merge|show` CLI with injectable `gh` runners.

**Tech Stack:** TypeScript (strict, ES2022, NodeNext), Zod, execa, vitest, Commander.

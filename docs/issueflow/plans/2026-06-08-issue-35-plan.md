# Candidate Branch Creation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `CandidateBranchIntegrator` to merge team/agent worktree branches into a provenance-tracked candidate branch, surfacing conflicts structurally without silent resolution.

**Architecture:** New `src/integration/` module with naming, Zod-validated provenance store (`candidate-branch.json` via `getIssueflowPath`), and git-backed `createCandidateBranch` with injectable `runGit`. Thin `issueflow candidate create|show` CLI in `src/commands/candidate.ts`. Isolation regression test guards all `src/workflow/**/*.ts` from importing integration code.

**Tech Stack:** TypeScript (strict, ES2022, NodeNext), Zod, execa, vitest, Commander.

  const rawPath = await getIssueflowPath(worktreePath, 'candidate-branch.json');

- Clean two-source merge → `status: 'created'`, `mergeCommitSha`, provenance `status: 'ready'`

- [ ] **Step 2: Run test — expect FAIL**

- [ ] **Step 3: Implement integrator**

Algorithm:
1. Validate sources non-empty
2. For each source: `git rev-parse --verify refs/heads/<branchName>`
3. `git checkout -B <candidateBranch> <baseBranch>`
4. Sequential `git merge --no-commit --no-ff <source>`
5. `git commit -m "candidate: integrate team <teamId> for issue #<issueNumber>"`
6. Write provenance

Export `defaultRunGit` wrapping `execa('git', args, { cwd, reject: false })` → `{ stdout, stderr, exitCode }`.

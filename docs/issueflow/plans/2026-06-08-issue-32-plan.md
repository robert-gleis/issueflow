# Session Replay Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship offline workflow session replay from persisted Event Log rows and agent log snapshots, with composition-root persistence helpers and `issueflow replay show`.

**Architecture:** New `src/replay/` module (log snapshot store migration v4, builder, formatters, persistence helpers). Extend `EVENT_TYPES` and `EventQuery.order`. Wire `registerReplayCommands` in CLI. No `src/workflow/` imports from replay.

**Trade-off:** `buildWorkflowReplay` uses `EventLog.list` with `limit: 1000` (store max). Replays for issues exceeding 1000 events are truncated in v1.

**Tech Stack:** TypeScript (strict, ES2022, NodeNext), better-sqlite3 via state-store, vitest, Commander.

  agentId: string;
  issueId: number;
  workflowId?: string;
  snapshot: AgentLogSnapshot;
  eventLog: EventLog;
  logStore: AgentLogStore;
}): void {
  const row = input.logStore.capture({ ... });

    eventType: 'agent.log.captured',
